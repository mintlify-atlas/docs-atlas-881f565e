# Quick Start Guide

This guide will help you get up and running with dotly quickly. By the end, you'll understand the basic workflow and be able to start customizing your development environment.

## Prerequisites

Before starting, make sure you have:

- dotly installed on your system ([Installation Guide](/installation))
- Restarted your terminal after installation
- A GitHub account (for backing up your dotfiles)

## Verify Installation

First, verify that dotly is properly installed:

```bash
dot -h
```

You should see the help message:

```
Usage:
   dot
   dot <context>
   dot <context> <script> [<args>...]
   dot -h | --help
```

<Info>
  If the `dot` command is not found, make sure you restarted your terminal after installation.
</Info>

## Understanding Your Dotfiles Structure

Your dotfiles are now located at `~/.dotfiles` (or wherever you specified during installation). Let's explore the structure:

```bash
cd ~/.dotfiles
ls -la
```

You'll see a structure like this:

```bash
├── bin                 # External binaries/symlinks (highest $PATH priority)
├── doc                 # Documentation for your dotfiles
├── editors             # Editor settings (VSCode, IntelliJ IDEA, Vim, etc.)
├── git                 # Git configuration
├── langs               # Programming language configs (Node, Python, Java, etc.)
├── os                  # OS-specific configs and apps
├── restoration_scripts # Scripts to run when restoring dotfiles
├── scripts             # Your custom scripts organized by context
├── shell               # Bash/Zsh configuration files
└── symlinks            # Symlink configuration files
```

Each directory serves a specific purpose in organizing your development environment.

## Create Your Dotfiles Repository

To back up and sync your dotfiles across machines, you need to push them to a Git repository.

<Steps>
  <Step title="Create a GitHub repository">
    1. Go to [GitHub](https://github.com/new)
    2. Create a new repository named `dotfiles`
    3. Make it private (recommended) or public
    4. Don't initialize with README, .gitignore, or license
    5. Copy the repository URL
  </Step>

  <Step title="Connect and push your dotfiles">
    Navigate to your dotfiles directory and push:
    
    ```bash
    cd ~/.dotfiles
    git remote add origin YOUR_REPOSITORY_URL
    git add -A
    git commit -m "Initial commit with dotly"
    git push -u origin main
    ```
    
    Replace `YOUR_REPOSITORY_URL` with your actual repository URL.
  </Step>
</Steps>

<Warning>
  Always commit your changes when you modify configs or add scripts. This ensures your dotfiles are backed up and can be restored on other machines.
</Warning>

## Using the `dot` Command

The `dot` command is the heart of dotly. It provides an interactive way to run scripts and manage your dotfiles.

### Interactive Mode

Run `dot` without arguments to see all available scripts:

```bash
dot
```

This opens an interactive fuzzy finder where you can:
- Browse all available scripts
- Preview script documentation
- Select and run scripts
- Pass arguments to scripts

### Context Mode

View scripts in a specific context:

```bash
# View all git-related scripts
dot git

# View all package management scripts
dot package

# View all self-maintenance scripts
dot self
```

### Direct Execution

Run a specific script directly:

```bash
# Run a git commit
dot git commit

# Update all packages
dot package update_all

# Debug dotly
dot self debug
```

## Exploring Built-in Scripts

dotly comes with many useful scripts out of the box. Here are some of the most commonly used:

### Git Scripts

```bash
# Add all files and commit
dot git commit

# Show a pretty git log with filtering
dot git pretty-log

# Show a pretty git diff (with fzf)
dot git pretty-diff

# Amend the last commit
dot git amend

# List all contributors
dot git contributors
```

### Package Management

```bash
# Add/install a package
dot package add

# Update all packages (brew, apt, etc.)
dot package update_all

# Export installed packages list
dot package dump

# Import previously dumped packages
dot package import
```

### Symlinks

```bash
# Apply all configured symlinks
dot symlinks apply
```

### Self Maintenance

```bash
# Update dotly to latest version
dot self update

# Debug dotly issues
dot self debug

# Run static analysis
dot self static_analysis
```

## Install Your First Package

Let's install a package using dotly's package management:

<Steps>
  <Step title="Add a package">
    ```bash
    dot package add
    ```
    
    This will prompt you to enter the package name (e.g., `jq`, `htop`, `tree`).
  </Step>

  <Step title="Verify installation">
    After installation, the package information is saved in your dotfiles and can be synced across machines.
  </Step>

  <Step title="Export your packages">
    Export all installed packages so you can restore them later:
    
    ```bash
    dot package dump
    ```
  </Step>

  <Step title="Commit the changes">
    ```bash
    cd ~/.dotfiles
    git add -A
    git commit -m "Add package: jq"
    git push
    ```
  </Step>
</Steps>

## Customize Your Shell

Let's customize your shell configuration:

### Add an Alias

Edit `~/.dotfiles/shell/aliases.sh` to add custom aliases:

```bash
# Navigate to aliases file
cd ~/.dotfiles
vim shell/aliases.sh  # or use your preferred editor
```

Add your custom aliases:

```bash
# Your custom aliases
alias projects="cd ~/projects"
alias serve="python -m http.server"
alias weather="curl wttr.in"
```

dotly already includes useful aliases:

```bash
# Directory navigation
..          # cd ..
...         # cd ../..
dotfiles    # cd to your dotfiles directory

# Git shortcuts
gaa         # git add -A
gs          # git status -sb
gps         # git push
gpl         # git pull --rebase --autostash

# Utils
up          # Update all packages
la          # ls -la with colors
```

### Configure Theme

Edit `~/.dotfiles/shell/exports.sh` to customize the terminal theme:

```bash
# Codely theme configuration
export CODELY_THEME_MINIMAL=false          # Set to true for minimal prompt
export CODELY_THEME_MODE="dark"            # "dark" or "light"
export CODELY_THEME_PROMPT_IN_NEW_LINE=false  # Prompt on new line
export CODELY_THEME_PWD_MODE="short"       # "short", "full", or "home_relative"
```

### Apply Changes

After making changes, reload your shell:

```bash
# Restart your shell
exec zsh

# Or source the config
source ~/.zshrc
```

## Version Control App Configurations

One of dotly's most powerful features is versioning application configurations. Here's how to version your VSCode settings:

<Steps>
  <Step title="Locate your config file">
    Find where your application stores its config. For VSCode on macOS:
    ```
    ~/Library/Application Support/Code/User/settings.json
    ```
  </Step>

  <Step title="Copy config to dotfiles">
    ```bash
    cp "~/Library/Application Support/Code/User/settings.json" \
       ~/.dotfiles/editors/code/settings.json
    ```
  </Step>

  <Step title="Configure symlink">
    Edit the appropriate symlink config file for your OS:
    
    ```bash
    # For macOS
    vim ~/.dotfiles/symlinks/conf.macos.yaml
    
    # For Linux
    vim ~/.dotfiles/symlinks/conf.linux.yaml
    ```
    
    Add your symlink:
    ```yaml
    ~/Library/Application Support/Code/User/settings.json: editors/code/settings.json
    ```
  </Step>

  <Step title="Apply symlinks">
    ```bash
    dot symlinks apply
    ```
    
    Now the file in your dotfiles is the source of truth, and changes sync automatically!
  </Step>

  <Step title="Commit to Git">
    ```bash
    cd ~/.dotfiles
    git add -A
    git commit -m "Add VSCode settings"
    git push
    ```
  </Step>
</Steps>

<Note>
  After setting up symlinks, any changes you make in VSCode settings will be reflected in your dotfiles repository.
</Note>

## Create Your First Custom Script

Let's create a simple custom script:

<Steps>
  <Step title="Create a script directory">
    Scripts are organized by context. Let's create a "work" context:
    ```bash
    mkdir -p ~/.dotfiles/scripts/work
    ```
  </Step>

  <Step title="Create a script file">
    ```bash
    vim ~/.dotfiles/scripts/work/standup
    ```
    
    Add the following content:
    ```bash
    #!/usr/bin/env bash
    
    ##? Opens today's standup notes
    ##?
    ##? Usage:
    ##?    standup
    
    today=$(date +%Y-%m-%d)
    notes_dir="$HOME/Documents/standup"
    notes_file="$notes_dir/$today.md"
    
    mkdir -p "$notes_dir"
    
    if [ ! -f "$notes_file" ]; then
      echo "# Standup - $today" > "$notes_file"
      echo "" >> "$notes_file"
      echo "## Yesterday" >> "$notes_file"
      echo "" >> "$notes_file"
      echo "## Today" >> "$notes_file"
      echo "" >> "$notes_file"
      echo "## Blockers" >> "$notes_file"
    fi
    
    ${EDITOR:-vim} "$notes_file"
    ```
  </Step>

  <Step title="Make it executable">
    ```bash
    chmod +x ~/.dotfiles/scripts/work/standup
    ```
  </Step>

  <Step title="Use your script">
    ```bash
    # Run interactively
    dot work
    # Select 'standup'
    
    # Or run directly
    dot work standup
    ```
  </Step>

  <Step title="Commit your script">
    ```bash
    cd ~/.dotfiles
    git add scripts/work/standup
    git commit -m "Add standup script"
    git push
    ```
  </Step>
</Steps>

<Info>
  The `##?` comments at the top of the script provide documentation that appears in the fuzzy finder preview.
</Info>

## Restore on Another Machine

When you get a new computer or want to sync your environment:

<CodeGroup>

```bash curl
bash <(curl -s https://raw.githubusercontent.com/CodelyTV/dotly/HEAD/restorer)
```

```bash wget
bash <(wget -qO- https://raw.githubusercontent.com/CodelyTV/dotly/HEAD/restorer)
```

</CodeGroup>

Provide your GitHub username and repository name, and your entire environment will be restored!

## Common Workflows

### Daily Usage

```bash
# Update all packages
up  # alias for 'dot package update_all'

# Navigate to dotfiles
dotfiles  # alias for 'cd $DOTFILES_PATH'

# Git operations
gs   # git status
gaa  # git add all
gc   # git commit (using dot git commit)
gps  # git push
```

### Adding New Configurations

```bash
# 1. Add or modify files in ~/.dotfiles
cd ~/.dotfiles
vim shell/aliases.sh

# 2. Commit changes
git add -A
git commit -m "Add new aliases"
git push

# 3. Reload shell
exec zsh
```

### Syncing Across Machines

```bash
# On machine A: Push changes
cd ~/.dotfiles
git add -A
git commit -m "Update configs"
git push

# On machine B: Pull changes
cd ~/.dotfiles
git pull
# Restart terminal or source ~/.zshrc
```
